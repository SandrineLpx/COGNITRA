# docs

Project documentation lives here.

## Structure

- `technical_report_v1.0.md`: main technical report for the project.
- `streamlit_page_map.md`: current Streamlit page map (routes, purpose, module ownership).
- `ITERATION_LOG.md`: concise milestone log.
- `ITERATION_LOG_FULL.md`: detailed iteration history.
- `UI/`: UI and UX documentation.
- `quality/`: quality standards, KPI definitions, and QC prompt specs.
- `reference/`: active reference material used for analyst workflows.
  - `company-watchlist.md`
  - `executive-brief-template.md`
  - `QUARTERLY_REVIEW.md`
  - `archive/`: historical or deprecated reference files kept for traceability.

## What is intentionally not public

The following are local-only and ignored from git:

- `docs/Assignment_info/`
- `docs/Demo/`
- `docs/Perso/`

These folders may contain coursework prompts, proprietary demo files, or personal working materials.
